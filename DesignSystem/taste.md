# Toopdbq — Taste ルール（"素人臭さ"を消す美学）

`WdTextButton` / `WdCircleCard` / `WdUserBar` の再設計から抽出した、**全コンポーネント共通の品位基準**。世界観（ダーク＋カラフル放射グラデ＋ガラス＋写真フルブリード＋Noto Sans JP/Inter）は維持したまま、"盛りすぎ・不揃い・強弱不足"を排除する。

> 一行で: **「効果は引き算、階層は明確、寸法はグリッド、可読性は構造で保証」**。

---

## 7原則（プロに見える根拠つき）

1. **1コンポーネント＝視覚効果1〜2個。** 枠・影・グラデを同時に盛らない。**liquid glass（面＋エッジ）は 1 効果と数える。**
   *根拠:* 効果が増えるほど"自信のなさ"に見える。主役の効果を1つ決め、他は捨てる。
2. **可読性は偶然(text-shadow)でなく構造(スクリム/面)で保証。** 写真の上は `--scrim-min`(黒45%)以上、グラデCTAは中央スクリムで白を担保。
   *根拠:* 影頼みは背景次第で破綻。プロは"必ず読める"状態を設計する。
3. **主役は1つ。脇役は `--text-2` / `--text-3` へ降格。** 1ブロックに同強度のテキストを2つ置かない。
   *根拠:* 視線の入口が1つに定まると"整って"見える。
4. **内側 padding は 8pt の一定比率。** 16 / 20 / 24 を使い、上下左右で不揃いにしない。
   *根拠:* 余白の不揃いは素人サインの筆頭。
5. **寸法はグリッドに正規化。** 47→44、15→12、9→8 など Figma 実値の端数を 4/8 グリッドへ寄せる。タップ域は最低 **44px**。
   *根拠:* 端数寸法は"目測で置いた"印象を与える。
6. **アイコンは 16 / 20 / 24 の3サイズ＋光学中心合わせ。** 視覚的に大きい字形(♥や●)は気持ち小さく、細い字形は気持ち大きく。
   *根拠:* 数値中央≠光学中央。プロは目で合わせる。
7. **角丸・影は radius / elevation スケールに正規化。** 7と8→8、99と100→9999。影は `--shadow-avatar/-card` 等の定義値のみ、ワンオフ禁止。
   *根拠:* バラバラな角丸・影は無意識に"安さ"を伝える。

## Liquid Glass 予算（LiquidGlassRefresh で追加 — 厳守）

liquid glass（`--lg-*` / `.lg`）は DS の標準表面だが、**1層ごとに背景テクスチャ生成が走る**。美学と同時に性能予算：

- **chrome 限定**: ナビヘッダー／フッター／サイドレール／シート面／単発ボタン／composer ピルだけ。
- **セル禁止**: リスト・グリッドのセル（チャット行・コメント行・フィード・リール）と毎フレーム動く層（Ken Burns・PageView 内部・3D 連動ピン）には置かない。セルは `--surface-raised` / scrim で。
- **≤6 形状／画面**。シートのような大面積も1枚と数える。
- ガラスの中で頻繁に更新されるのは**テキストのみ**（例: like カウント）。ガラス形状自体をアニメさせない。
- 各 specimen に **`glass: yes / no(perf)`** を明記（`.glass-tag`）。迷ったら no。

## Smell test（出す前のチェック）

- [ ] 効果（枠/影/グラデ/グロー）を**3つ以上**盛っていないか？ → 1〜2に削る（liquid glass は1つと数える）
- [ ] glass を chrome 以外（リストセル・動く層）に置いていないか？ 1画面の glass 形状は **≤6** か？
- [ ] 写真の上の文字に**スクリムor面**があるか？ text-shadowだけになっていないか
- [ ] このブロックの**主役は1つ**か？ 同じ強さの白文字が並んでいないか
- [ ] padding は **8pt(16/20/24)** か？ 上下左右で揃っているか
- [ ] 寸法に **47/15/9** のような端数が残っていないか
- [ ] タップ域 **≥44px** か
- [ ] 角丸/影は**スケールの値**か（7/99/ワンオフ影が混ざっていないか）
- [ ] 不要な **中央寄せ** をしていないか（左揃えが自然な情報を中央に置いていないか）
